# Community Board Trunk

This folder is the active growth surface for trunk-first cutover work.

## Structure

- `soil/` raw practitioner and community intake
- `fibers/` append-first operational records
- `tags/` derived and stabilized tags
- `overnight/` scheduled growth jobs and outputs
- `branches/` read-model outputs consumed by app surfaces
- `heartwood/legacy-pointers/` references to previous active rings
- `config/` trunk-governed runtime policies

## Cutover rule

No delete.
Divide and grow new.
Legacy paths remain as heartwood pointers while net-new writes move to trunk surfaces.
