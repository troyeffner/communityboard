# Human Update Style Policy

Status: canonical
Owner: QB

## Purpose
Make QB updates understandable in plain language first, with logistics second.

## Required Response Order
1. What We Learned
2. What Changed
3. What Matters Next
4. Evidence/Logistics (only if needed)

## Rules
- Do not lead with file lists, command logs, or path dumps.
- Explain impact in practical language.
- Keep updates concise and decision-oriented.
- Use file paths only as supporting evidence.

## Applies To
- QB check/update responses
- Progress updates after processing requests
- Daily report summaries in chat
