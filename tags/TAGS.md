# Tags

Tags are declared synapses. They are specific to the fractal, assigned by practitioners, and variable. They are how a fractal finds and is found within its family tree.

## What Tags Do

Tags connect this node to knowledge inside its own boundary. Family. The tree it belongs to. What the nodes around it have learned and are learning. Tags add specificity on top of the baseline that ports establish.

## What Lives Here

Tag registries, indexes, and schema definitions for this node. Tags declared here apply to fibers, files, and fractals within this node's scope.

The canonical tag dimensions (declared synapses) are:
  1. Layer: where in the tree
  2. Type: what function in the flow
  3. Project: which product tree
  4. Practitioner: who acts on this
  5. Lifecycle: decompose / absorb / produce
  6. Cluster Stability: forming / stable / hardening / soil

Full schema reference: divergent-networks-trunk/ops-hub/tags/SCHEMA.md

## Relationship to Ports

Ports learn from society. Tags learn from family. Both exist at every level of the fractal. Same structure, same purpose, all the way down. The distinction is direction of learning, not location in the tree.
